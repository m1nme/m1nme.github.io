# 文件上传漏洞简介

文件上传漏洞是 web 安全中经常遇到的一种漏洞形式，文件上传漏洞就是利用可以上传文件的地方将恶意代码或文件植入到服务器中，再通过 URL 去访问来执行代码，但是经常会遇到各种限制和过滤，因此就要学会各种姿势来绕过。

# 常见的限制以及绕过

# 客户端验证（前端验证)

# 原理

就是通过前端代码来检查用户所上传的文件的扩展名，对于攻击者来说是极其容易绕过的，因此更多是用于防止用户错误上传文件类型的措施。

# 绕过

  1. 先上传一个.gif 或者.JPG 之类的木马，突破前端验证后再通过抓包修改文件后缀名来实现绕过
  2. 通过禁用或者删除 JS 来实现突破

# 服务端验证

# MIME 类型检测

# 原理

服务端通过检测 MIME 来限制文件上传的类型，其代码可能如下：
$_FILES['flie']['type'] == 'image/jpeg'

# 绕过

通过抓包修改 Content-Type

# 检查文件内容

# 原理

检查文件内容是否包含 “<?”、“<?= ” 之类的关键字

# 绕过

可以使用短 tag 绕过，例如
<script language='pHp'>phpinfo();</script>

# 检查文件名

# 原理

检查文件名是否包含.php

# 绕过
  1. 大小写绕过，如.PhP
  2. 使用其他可能被 APache 解析的后缀，如（.php4/.php5/.php7/.pht/.phtml）
  3. 在 Apache 环境下上传.htaccess 文件使其它后缀也被解析为 php 脚本执行。.htaccess 文件内容为：
    AddType application/x-http-php .jpg
  4. %00 截断，操作方法：上传 shell.jpg，Burp 抓包，将文件名改为 shell.php%00.jpg，选中 %00，进行 url-decode。
